# ExpenseSheetManagement
Project Description

Project to build a browser application where users can enter expense sheets and download reports of their expenses. Users can submit an expense sheet for a specific date or enter an expense sheet with multiple lines. Receipts are mandatory and can be attached per expense line.

Project contents
  • Multiple users can access the application and enter expense sheets.
  • Users can download an expense report in csv where the report will use date parameters to filter data.
  • Users can bulk upload or downloaded expenses using REST API.
Key topics covered in project
  • Object oriented programming.Using Java OOP concepts to build the project.
  • Test driven development.Using TDD to create tests and then build code while refactoring along the way. Unit and integration tests provided.
  • REST API.Using postman to create data with json file specs provided.
  • Database Management.Storing data for each expense and expense line against a user record while being able to pull the data stored for each user.
